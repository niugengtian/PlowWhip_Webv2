# Web v2 MVP release notes

This release replaces the Desktop-dependent prototype with an independent local Web product. It provides verified tasks, multi-project role workers, durable CLI session identity, one OS scheduler, zero-token control paths, bounded context, three-scope Convention, token accounting, rotation, recovery, permissions, audit and complete operating views.

MVP intentionally does not promise SaaS hosting, multi-machine clustering, hostile-code sandboxing or compatibility with every CLI provider. Generic Command is production-testable locally. Codex, Cursor and Claude entries remain visibly unavailable until their adapters and credentials are configured; the system never disguises that state as completion.
